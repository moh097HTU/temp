System Architecture & Dataflow

Bench validation mode (plane on table): When tracking starts, Jetson streams Offboard attitude setpoints to PX4 to verify control surfaces move correctly (no flight dynamics).
  • PX4: v1.16.0 Stable Release
  • Offboard contract: stream SET_ATTITUDE_TARGET @ 30 Hz, thrust=0 (bench), roll/pitch from vision errors
  • MAVLink hub: MAVProxy runs on Jetson, owns TELEM1 (serial 57600), forwards MAVLink to Jetson apps + GCS PC
  • Custom QGC fork: runs on GCS computer like normal QGC; receives MAVLink over UDP and video over UDP (GStreamer)
  • ESP32: battery switch signals via GPIO into Jetson, injected into MAVLink for dashboard display in QGC

Bench Offboard “contract” (human-readable)

This is the operational agreement between your Jetson stack and PX4.
  • Rate: send attitude setpoints at 30 Hz continuously while tracking is active.
  • Bench mode: set thrust = 0 always.
  • Mapping: yaw_error → roll_sp, (optional) pitch_error → pitch_sp, range_error → throttle (disabled on bench).
  • Limits: roll_sp clamp (e.g., ±20°), pitch_sp clamp (e.g., ±10°), rate-limit setpoint changes.
  • Gating: only send non-zero setpoints if target lock is valid and detection is fresh; otherwise send neutral (0,0).
  • Failsafe: if tracking stream/lock is stale → neutral setpoints immediately; optionally command mode exit.
